
# HTML & CSS TIL 

## 1) 웹(Web)부터 감 잡기

웹은 “인터넷에 연결된 컴퓨터들이 정보를 공유하는 거대한 정보 공간”이다. 우리가 브라우저로 검색하고 클릭하고 로그인하고 글을 쓰는 모든 활동이 웹 위에서 일어난다. 

여기서 용어를 깔끔히 정리하면:

* **Web**: 웹사이트/웹앱을 통해 사용자가 정보를 검색하고 상호작용하게 해주는 기술/플랫폼 
* **Web site**: 여러 개의 **Web page**가 모여 사용자에게 정보/서비스를 제공하는 공간 
* **Web page**: HTML/CSS 같은 웹 기술로 만들어진 “페이지 한 장” (웹사이트를 구성하는 요소) 

자료의 그림(집 비유)이 핵심을 잘 보여준다. 웹 페이지는 크게 3요소로 생각하면 이해가 쉽다. 

* **구조(Structure)**: 뼈대, 기둥, 벽 → HTML
* **스타일(Styling)**: 페인트, 장식, 배치 → CSS
* **동작(Behavior)**: 스위치 켜기/끄기, 상호작용 → JavaScript

즉,

> HTML은 “무엇이 어디에 있는지”, CSS는 “어떻게 보이는지”, JS는 “어떻게 동작하는지”를 맡는다.

## 2) HTML: 웹 페이지의 “의미와 구조”를 만드는 언어

자료에서 HTML을 “웹 페이지의 의미와 구조를 정의하는 언어”라고 정의한다. 

### 2-1) HTML이 HyperText / Markup Language인 이유

* **HyperText**: 페이지와 페이지를 연결하는 링크(하이퍼링크)라는 개념이 핵심이라서 “HyperText”라고 부른다. 
* **Markup Language**: 태그(tag)로 문서 구조를 표시(markup)하는 언어다. 사람도 읽기 쉬운 형태이며 “의미/구조”에 집중한다. 

HTML은 “디자인을 예쁘게” 하는 언어가 아니다.
HTML은 **문서가 어떤 의미인지**를 태그로 표현해서 구조를 세운다.

---

## 3) HTML 기본 구조: 문서의 뼈대가 되는 4가지

HTML 문서는 크게 이런 뼈대를 가진다. (자료의 구조 예시) 

### 3-1) `<!DOCTYPE html>`

* “이 문서는 HTML5 문서입니다”라고 브라우저에게 선언하는 줄이다. 

### 3-2) `<html> ... </html>`

* HTML 문서의 최상위(root) 요소다.
* 일반적으로 `lang="en"` 같은 언어 정보도 여기에 붙인다. 

### 3-3) `<head> ... </head>`

* 사용자 화면에 직접 보이는 내용이 아니라, **문서에 대한 정보**를 담는다. 
* 대표적으로:

  * 문자 인코딩 `<meta charset="UTF-8">`
  * 문서 제목 `<title> ... </title>`
  * CSS 파일 연결 `<link ...>`
  * 검색/공유용 메타 태그 등

### 3-4) `<body> ... </body>`

* 사용자 화면에 실제로 표시되는 내용을 담는다. 
* 우리가 만드는 화면의 대부분은 body 안에 들어간다.

## 4) HTML Element(요소), Tag(태그), Content(내용)의 관계

자료에서는 HTML Element가 “여는 태그 + 닫는 태그 + 내용”으로 구성된다고 설명한다. 

예를 들어:

```html
<p>My cat is very grumpy</p>
```

* `<p>` : 여는 태그(opening tag)
* `</p>` : 닫는 태그(closing tag)
* `My cat is very grumpy` : 내용(content)
* 전체가 합쳐져서 **하나의 요소(element)** 가 된다.

HTML을 볼 때는 “태그를 해석”하는 게 아니라,
“요소들이 트리 구조로 중첩되어 문서 구조를 만든다”고 이해하는 게 중요하다.

---

## 5) HTML Attributes(속성): 요소에 “추가 정보”를 붙이는 법

속성은 태그에 붙어서 요소의 동작/식별/설정을 돕는다. 자료에선 예시로 `class` 같은 속성을 보여준다. 

예:

```html
<p class="editor-note">My cat is very grumpy</p>
```

* `class="editor-note"` 가 속성(attribute)
* 속성은 보통 이런 용도로 쓰인다.

  * CSS 적용 대상 지정 (class, id)
  * 링크 이동(href), 이미지 경로(src)
  * 입력값 제어(type, placeholder) 등

---

## 6) HTML Text Structure: “텍스트도 구조가 있다”

자료에서 강조하는 포인트는 이거다:
**텍스트를 크게/작게 보여주려고 h1~h6를 쓰는 게 아니라, 의미에 맞게 써야 한다.** 

자주 쓰는 텍스트 구조 태그:

* **Heading**: `<h1>` ~ `<h6>`
* **Paragraph**: `<p>`
* **List**:

  * `<ul>` (순서 없는 리스트)
  * `<ol>` (순서 있는 리스트)
  * `<li>` (리스트 항목)
* **Emphasis / Important**:

  * `<em>`: 강조(기울임처럼 보이지만 “강조” 의미)
  * `<strong>`: 중요(굵게 보일 수 있지만 “중요” 의미) 

정리하면, HTML은 “보이는 모양”보다 “의미”를 먼저 잡는 게 원칙이다.
모양은 CSS가 담당한다.

# 7) CSS: 웹 페이지의 “디자인과 레이아웃”을 구성하는 언어

CSS는 자료에서 “웹 페이지의 디자인과 레이아웃을 구성하는 언어”라고 정의한다. 

HTML이 구조를 만들면, CSS는 그 구조에 스타일을 입힌다.

---

## 8) CSS 적용 방법 3가지: Inline / Internal / External

자료에 3가지가 나온다. 각각 장단점이 있고, 실무에선 보통 **External**이 기본이다. 

### 8-1) Inline 스타일 (HTML 태그에 직접)

```html
<p style="color: blue; background-color: yellow;">Hello</p>
```

* 장점: 빠르게 한 요소만 테스트할 때 간편
* 단점: 유지보수 최악(스타일이 HTML에 섞여버림)

### 8-2) Internal 스타일 (head 안 `<style>`)

```html
<head>
  <style>
    p { color: blue; }
  </style>
</head>
```

* 장점: 한 파일 안에서 해결 가능
* 단점: 페이지가 많아지면 관리가 어려움

### 8-3) External 스타일 (별도 CSS 파일 + link)

```html
<link rel="stylesheet" href="style.css">
```

* 장점: 유지보수/재사용 최강
* 단점: 파일이 분리되지만, 그게 오히려 “정석”이다. 

---

## 9) CSS 구문: Selector + Declaration Block

자료는 CSS의 기본 구조를 다음처럼 설명한다. 

```css
h1 {
  color: red;
  font-size: 30px;
}
```

* `h1` : **선택자(selector)** → “누구에게 적용할까?”
* `{ ... }` : **선언 블록(declaration block)**
* `color: red;` 같은 한 줄이 **선언(declaration)**
* 선언은

  * `color` : **속성(property)**
  * `red` : **값(value)**
    로 구성된다. 

---

## 10) CSS 선택자(Selectors): “어떤 요소를 찍을지”가 실력

자료에서는 선택자를 크게 “기본 선택자”와 “결합자(combinators)”로 소개한다. 

### 10-1) 기본 선택자

* **전체 선택자**: `*` (모든 요소)
* **요소 선택자**: `p`, `h1` 처럼 태그 이름
* **클래스 선택자**: `.className`
* **아이디 선택자**: `#idName`
* **속성 선택자**: `[type="text"]` 같은 형태(개념만 기억)

클래스와 id의 핵심 차이:

* `id`는 “문서에서 유일”해야 하는 식별자 성격
* `class`는 “여러 요소에 반복 적용”하는 그룹 성격
  → 보통 스타일링은 class 중심이 더 자연스럽다.

### 10-2) 결합자(Combinators)

자료에 나온 대표 결합자: 

* **자손 결합자(공백)**: `A B`

  * A 안에 있는 모든 B(깊이 상관없이)
* **자식 결합자(>)**: `A > B`

  * A의 “바로 아래 1단계 자식” B만

예를 들어:

```css
div p { color: red; }   /* div 안의 모든 p */
div > p { color: blue; } /* div 바로 아래 자식 p만 */
```

---

## 11) CSS 값 단위(Unit): px, em, rem 핵심

자료에서 단위를 비교해서 설명한다. 

### 11-1) px (절대 단위처럼 쓰는 대표 단위)

* 픽셀 기반으로 직관적이라 많이 쓰지만,
* 사용자 환경(디스플레이/접근성 설정)에 따라 유연성이 떨어질 수 있다.

### 11-2) em (상대 단위)

* **부모 요소의 font-size**를 기준으로 크기가 결정되는 상대 단위
* 중첩되면 계산이 복잡해질 수 있다.

### 11-3) rem (root em)

* **최상위(html)의 font-size**를 기준으로 계산되는 상대 단위 
* em의 중첩 문제를 줄여서 전체적인 일관성 관리가 좋다.

실무에서는 폰트/간격을 rem 기반으로 설계하는 경우가 많다.
(“전체 스케일을 root font-size로 통제”하기 쉬워서)

# 12) CSS 명시도(Specificity) & Cascade: “왜 내 스타일이 적용이 안 되지?”의 답

자료에서 **명시도**는 “어떤 CSS 선언이 최종 적용될지 결정하는 알고리즘”이라고 말한다. 

여기서 CSS 이름이 Cascading Style Sheet인 이유가 나온다.
**여러 규칙이 겹치면 우선순위를 계산해서 하나가 이긴다.** 

우선순위를 이해할 때 핵심은 4단계다(자료의 요지): 

1. `!important` (최후의 수단, 웬만하면 남용 금지)
2. **인라인 스타일** (태그에 직접 적은 style)
3. **명시도 점수** (선택자 강함)
4. **선언 순서** (동일하면 “나중에 적힌 것”이 이김)

명시도 감각(기본 규칙):

* `#id`가 `.class`보다 강하다
* `.class`가 `tag`보다 강하다
* 구체적으로 찍을수록 강해진다

그래서 어떤 요소에 스타일이 안 먹으면 보통 이런 이유다:

* 더 강한 선택자가 이미 덮어쓰고 있다.
* 같은 선택자인데 아래에 적힌 규칙이 덮어썼다.
* 인라인/important가 끼어 있다.

---

## 13) 상속(Inheritance): “부모 스타일이 자식에게 내려오는가?”

자료에서 CSS는 기본적으로 상속을 통해 재사용성을 높인다고 설명한다. 

### 13-1) 상속되는 속성(대표)

* 텍스트 관련: `font`, `color`, `text-align`
* `opacity`, `visibility` 등 

즉, 부모에 `color: red`를 주면 자식 텍스트도 빨개지는 게 자연스럽다.

### 13-2) 상속되지 않는 속성(대표)

* 박스 모델 관련: `width`, `height`, `border`, `box-sizing` …
* 위치 관련: `position`, `top/right/bottom/left`, `z-index` … 

즉, 부모의 border가 자식에게 “상속”되면 이상하니까 상속이 안 되는 게 기본이다.

# 14) CSS Box Model: 웹의 모든 요소는 “사각형 상자”다

자료에서 가장 중요하게 다루는 주제 중 하나가 Box Model이다. 

> 웹 페이지의 모든 HTML 요소는 “사각형 상자”로 취급되고,
> 레이아웃은 이 상자들의 크기/간격/배치로 결정된다.

박스는 다음 구성요소로 이루어진다(자료 도식): 

1. **Content**: 실제 내용 영역(텍스트/이미지)
2. **Padding**: 내용과 테두리 사이의 안쪽 여백
3. **Border**: 테두리
4. **Margin**: 바깥 여백(다른 요소와의 거리)

### 14-1) Margin vs Padding 헷갈림 정리

* **Padding**: “내 상자 안쪽” 여백
  → 배경색(background)이 padding까지 칠해지는 게 보통이다.
* **Margin**: “내 상자 바깥” 여백
  → 다른 요소와의 간격을 만든다.

---

## 15) 단축 속성(shorthand): margin/padding/border를 한 줄로

자료에 단축 속성이 나온다. 

대표:

```css
margin: 10px 20px 30px 40px; /* 위 오른쪽 아래 왼쪽 */
padding: 10px 20px;          /* 위아래 / 좌우 */
border: 2px solid black;     /* 두께 스타일 색상 */
```

* 특히 `margin/padding`의 값 개수에 따른 규칙은 자주 쓰이니 외워두면 좋다.

---

## 16) box-sizing: “width/height가 어디까지를 의미하냐” 문제

자료에 standard box model vs alternative box model 그림이 나온다. 

### 16-1) 기본값(content-box)

```css
box-sizing: content-box;
```

* `width`, `height`는 **content 영역만** 의미
* padding/border가 추가되면 실제 박스 크기는 더 커진다
  → 레이아웃 짤 때 “생각보다 박스가 커지는” 문제의 원인

### 16-2) border-box (실무에서 자주 쓰는 방식)

```css
box-sizing: border-box;
```

* `width`, `height`에 **padding + border까지 포함**
* 즉, “지정한 width 안에서 padding/border가 안쪽으로 들어간다”
* 결과적으로 레이아웃이 훨씬 예측 가능해진다. 

그래서 프로젝트 시작할 때 이런 식으로 전체에 적용하는 패턴이 흔하다:

```css
* { box-sizing: border-box; }
```

# 17) 스타일 가이드 & MDN: 공부할 때 반드시 필요한 습관

자료 후반에는 HTML/CSS 스타일 가이드와 MDN이 나온다. 

여기서 핵심은 두 가지:

1. **스타일 가이드는 “협업과 유지보수”를 위한 약속**

   * 태그/속성 작성 규칙, 들여쓰기, 네이밍, 공백 규칙 등
   * “코드가 예쁘게 보이게”를 넘어, 팀에서 충돌을 줄인다.

2. **MDN은 표준 문서의 핵심 레퍼런스**

   * HTML/CSS/JS 웹 표준을 정리한 문서
   * “이 속성 뭐였지?” “이 값 가능한가?” 같은 걸 확인할 때 가장 신뢰도가 높다.

# 18) 오늘 내용이 실전에서 어떻게 이어지나?

이 HTML/CSS 기초는 앞으로 Vue 같은 프레임워크를 배울 때 이렇게 연결된다.

* Vue 템플릿(`<template>`)은 결국 HTML 구조를 더 편하게 작성하는 방식이다.
* Vue 스타일(`<style scoped>`)은 CSS 스코프를 컴포넌트 단위로 관리하는 방식이다.
* 레이아웃이 이상하면 대부분 Box Model / box-sizing / margin/padding에서 원인을 찾게 된다.
* “왜 CSS가 적용이 안 되지?”는 대개 **명시도 + cascade** 문제다.

즉, 프레임워크를 쓰더라도 결국 밑바닥에는 HTML/CSS 원리가 그대로 작동한다.

# ✅ 요약

* 웹페이지 구성: **HTML(구조) + CSS(디자인) + JS(동작)** 
* HTML: 의미와 구조를 정의하는 마크업 언어(태그, 요소, 속성) 
* CSS: 선택자(selector)로 대상 지정 → 선언(property:value)으로 스타일 적용 
* 선택자: `*`, 태그, `.class`, `#id`, 결합자(자손/자식) 
* 단위: `px`(직관), `em`(부모 기준), `rem`(root 기준) 
* 우선순위: `!important` > inline > specificity > 나중 선언 
* 상속: text 계열은 상속, box/position 계열은 보통 상속 X 
* 박스 모델: content/padding/border/margin, `box-sizing: border-box`로 예측 가능한 크기 
* MDN: 웹 표준 레퍼런스 문서로 습관적으로 찾아보기
