
# Responsive Web TIL

## 0) 반응형 웹을 왜 해야 하나

반응형 웹(Responsive Web Design)은 **디바이스 종류나 화면 크기와 상관없이**, 어디서든 **일관된 레이아웃과 사용자 경험**을 제공하는 디자인 기술이다.

요즘 사용자는 데스크탑(큰 화면)에서만 웹을 보지 않는다. 노트북, 태블릿, 모바일, 심지어 폴더블까지 화면 폭이 제각각이다. 같은 페이지라도 화면 폭이 달라지면:

* 글자가 너무 작거나 너무 커 보이고
* 카드가 한 줄에 너무 많이 들어가서 답답하거나
* 반대로 요소가 한 줄에 하나씩만 떨어져서 스크롤이 과도해지고
* 버튼이 너무 붙어서 누르기 힘들어지는 문제가 생긴다

반응형 웹은 이런 문제를 해결하기 위해 “화면 크기에 따라 레이아웃을 자연스럽게 재배치”한다.

---

# 1) Bootstrap Grid System: 반응형 레이아웃을 만드는 대표 도구

자료는 Bootstrap Grid system을 **웹페이지 레이아웃을 조정하는 12개의 컬럼으로 구성된 시스템**이라고 설명한다. 

핵심 개념은 아주 단순하다.

> 화면을 “보이지 않는 12칸 격자”로 나눈 다음,
> 각 요소가 몇 칸을 차지할지 정해서 배치한다.

예를 들어 한 줄(row)을 12칸이라고 생각하면:

* `6칸 + 6칸` → 반반 레이아웃
* `4칸 + 4칸 + 4칸` → 3등분 카드
* `3칸 + 9칸` → 사이드바 + 본문

이렇게 “분수처럼” 설계할 수 있어서 레이아웃이 규칙적이고 유지보수가 쉬워진다.

---

# 2) Grid system 구조: Container → Row → Column

자료의 그림(페이지 2~3)에 그리드 시스템의 구성요소가 크게 3개로 정리되어 있다. 

## 2-1) Container: 콘텐츠의 ‘최대 폭’을 잡는 틀

Container는 전체 콘텐츠를 감싸는 큰 박스다.
역할은 **좌우 여백을 확보하고**, 화면이 너무 넓어졌을 때 콘텐츠 폭을 적절히 제한해서 가독성을 지키는 것이다. 

감각적으로는:

* container가 “웹페이지의 작업대”
* 그 위에 row/col로 그리드를 올린다

## 2-2) Row: 한 줄(행)을 만드는 단위

Row는 “열(column)들을 담는 한 줄”이다.
Grid 시스템에서 컬럼들은 무조건 row 안에서 배치된다. 

중요한 이유:

* row는 기본적으로 내부 col들의 정렬을 위한 규칙(마진/패딩 구조)을 가진다
* “한 줄에 12칸”이라는 기준도 row 단위에서 성립한다

## 2-3) Column: 실제 콘텐츠가 들어가는 칸

Column은 row 안에서 “몇 칸을 차지할지”를 정해 콘텐츠를 배치하는 단위다. 

Bootstrap에서는 보통 `col-숫자` 형태로 쓰며, 숫자는 1~12까지 가능하다.
한 row 안에서 col들의 합이 12를 기준으로 배치된다고 보면 된다.

예시 감각:

* `col-4`는 12칸 중 4칸(= 1/3)을 차지
* `col-6`는 1/2
* `col-3`는 1/4

자료의 실습 코드(페이지 2~3)도 결국 `container → row → col-*` 구조를 반복해서 보여준다. 

---

# 3) “왜 하필 12칸?”: 12는 나누기 쉬운 숫자다

12는 1,2,3,4,6,12로 잘 나뉘기 때문에 레이아웃 설계가 편하다.

* 2분할: 6+6
* 3분할: 4+4+4
* 4분할: 3+3+3+3
* 1:2 비율: 4+8 또는 3+9
* 1:3 비율: 3+9 / 2+10 등도 가능

이 덕분에 “디자인이 바뀌어도” 규칙적으로 재배치하기가 쉽다.

---

# 4) 중첩(Nesting): column 안에 또 row/col을 넣어 더 정교하게

자료는 “하나의 column에도 또 다른 row를 넣을 수 있다”는 방식으로 **중첩(nesting)**을 설명한다(페이지 4~5).

핵심 규칙:

* **col 안에 row를 넣고**
* 그 row 안에 다시 col을 넣는다

즉, “큰 그리드 안에 작은 그리드를 다시 만드는 방식”이다.

왜 쓰냐면:

* 카드 영역을 크게 잡고 그 안에서

  * 이미지 영역(예: 4칸)
  * 텍스트 영역(예: 8칸)
    처럼 내부를 다시 나누고 싶을 때
* 상세 페이지에서

  * 왼쪽: 정보 영역
  * 오른쪽: 관련 콘텐츠
    같은 복합 구조를 만들 때

중첩을 쓸 때 기억할 점:

> 중첩된 row에서도 “그 row 기준으로 다시 12칸”이 시작된다.

바깥에서 col-8을 썼다고 해서 안쪽이 8칸 기준이 되는 게 아니라,
안쪽 row는 **또 12칸 체계**로 새로 계산된다고 생각해야 한다.

---

# 5) 상쇄(Offset): “왼쪽에 빈칸을 만들어 밀기”

자료는 offset을 “컬럼을 상쇄(밀어내기)”하는 방식으로 설명한다(페이지 5). 

쉽게 말하면:

> 왼쪽에 빈 공간(빈 컬럼)을 만든 것처럼 효과를 내서
> 특정 col을 오른쪽으로 이동시키는 기능

예를 들어:

* 한 줄 12칸 중

  * col-4짜리 박스를 “가운데쯤” 두고 싶으면
  * 왼쪽에 offset을 주어 위치를 조정한다

이게 유용한 상황:

* 로그인 폼/카드 같은 작은 요소를 화면 가운데로 맞추고 싶을 때
* 카드가 1개만 있는 줄에서 “왼쪽 정렬이 아니라 적당히 밀어서” 균형을 맞추고 싶을 때
* “반응형에서 특정 크기 이상일 때만 살짝 오른쪽으로 밀기” 같은 디자인 디테일

---

# 6) Gutters: 컬럼 사이 간격(여백) 제어

자료는 gutters를 “grid system에서 column 사이의 여백 영역”이라고 설명한다(페이지 6~7). 

이 간격이 왜 중요하냐면:

* 카드들이 너무 붙어 있으면 답답하고
* 너무 벌어지면 화면이 텅 빈 느낌이 나며
* 모바일에서 간격이 크면 화면 낭비가 된다

즉, gutters는 “카드 사이 숨 쉬는 공간”을 조절하는 장치다.

자료에서는 `g-*`, `gx-*`, `gy-*` 같은 형태로 간격을 조절하는 실습 흐름이 나온다(페이지 6~7). 

* `g-*` : 가로+세로 간격을 동시에
* `gx-*` : 가로(좌우) 간격만
* `gy-*` : 세로(위아래) 간격만

---

# 7) Breakpoints: 화면 폭에 따라 “몇 칸을 먹을지”를 바꾼다

자료는 반응형 그리드를 위해 **breakpoints**를 소개한다(페이지 7~8). 

Breakpoint는 한 문장으로:

> 특정 화면 너비 구간을 기준으로 레이아웃 규칙을 바꾸는 “경계값”

예를 들어:

* 모바일에서는 카드 1열(한 줄에 하나)
* 태블릿에서는 2열
* 데스크탑에서는 3~4열

이런 “자연스러운 재배치”가 breakpoints로 구현된다.

자료 표(페이지 8)는 Bootstrap에서 주로 쓰는 breakpoint 이름들이 나오고, 화면 크기에 따라 container 폭/적용 구간이 달라진다는 걸 보여준다. 

실전 감각으로는 이렇게 이해하면 된다:

* `xs` 같은 가장 작은 구간은 “기본값”
* `sm`, `md`, `lg`, `xl`, `xxl`로 갈수록 더 큰 화면을 의미
* `col-md-6` 같은 표현은
  “md 이상에서는 6칸(절반), md 미만에서는 기본 규칙대로(보통 한 줄 꽉 차게)”처럼 동작한다고 보면 된다

자료의 breakpoints 실습(페이지 8)은 “화면이 커질수록 col 비율을 바꾸면서 배치가 어떻게 달라지는지”를 연습하도록 구성되어 있다. 

---

# 8) Grid System은 결국 “Media Query의 설탕 코팅”이다

자료의 ‘CSS Layout 종합정리’ 부분(페이지 9)에서 핵심 메시지가 나온다. 

* Bootstrap Grid system은 **12칸 grid + breakpoints**로 반응형 레이아웃을 만든다
* CSS의 근본 도구는 **Media Query**이고,
* Bootstrap은 그 복잡한 작업을 “클래스 조합”으로 쉽게 쓰게 해준 것이다

즉, Bootstrap을 쓰면:

* 직접 media query를 길게 쓰지 않아도
* `col-sm-*`, `col-md-*` 같은 클래스만으로
* 반응형 레이아웃을 빠르게 만들 수 있다

# 9) UX & UI: 반응형은 ‘예쁘게’보다 ‘쓰기 편하게’가 목표

자료 후반(페이지 10~12)은 UX/UI를 정리한다. 

## 9-1) UX(User Experience)

UX는 “제품/서비스를 사용하는 사람이 느끼는 전체 경험과 만족도를 개선하고 최적화하는 분야”다. 

반응형과 UX가 연결되는 이유:

* 모바일에서 버튼이 작으면 “누르기 어려움” → 경험이 나빠짐
* 폰에서 글이 너무 작거나 줄이 너무 길면 “읽기 불편함” → 이탈
* 카드가 과하게 밀집하면 “정보 탐색이 어려움” → 피로감

즉, 반응형은 단순히 레이아웃을 맞추는 게 아니라 **사용자의 상황(디바이스/환경)에 맞춰 경험을 최적화**하는 UX 작업이기도 하다.

## 9-2) UI(User Interface)

UI는 “서비스와 사용자 간 상호작용을 가능하게 하는 디자인 요소들을 개발/구현하는 분야”다. 

UI 관점에서 반응형은:

* 같은 기능이라도 화면에 맞게 배치/크기/간격이 달라져야 “조작하기 좋은 UI”가 된다
* UI 요소(버튼/입력창/카드)의 규칙적 배치는 사용자가 학습하기 쉬운 인터페이스를 만든다

자료에서도 UI 예시를 들며 사용자와의 상호작용 요소로 설명한다. 

---

# 10) 오늘 내용 한 장 요약

* **반응형 웹**: 디바이스/화면 크기와 상관없이 일관된 레이아웃과 UX 제공 
* **Bootstrap Grid System**: 12 컬럼 기반 레이아웃 시스템 
* **기본 구조**: `container → row → col` 
* **Nesting(중첩)**: col 안에 row/col을 다시 넣어 더 정교한 레이아웃 
* **Offset(상쇄)**: 왼쪽에 빈칸을 만든 것처럼 col을 오른쪽으로 밀기 
* **Gutters**: 컬럼 사이 간격(여백) 제어, `g/gx/gy`로 조절 
* **Breakpoints**: 화면 폭 구간에 따라 col 비율/배치를 바꾸는 기준점 
* **Grid는 Media Query 기반**: Bootstrap은 이를 클래스 조합으로 쉽게 쓰게 해줌 
* **UX/UI 연결**: 반응형은 보기 좋게가 아니라 “환경에 맞게 쓰기 편하게” 만드는 것
