## 1. 컬렉션 전체 그림 먼저 잡기

파이썬에서 여러 값을 한 번에 관리하는 자료들을 **컬렉션(Collection)**이라고 생각하면 편하다.

* **시퀀스 계열**

  * `str`, `list`, `tuple`, `range`
  * 특징: 순서가 있고, 인덱스로 접근 가능, 슬라이싱 가능(일부는 불변)
* **비시퀀스 계열**

  * `dict`, `set`
  * 특징: 순서보다 “키-값” 혹은 “집합” 개념이 더 중요

또 한 가지 큰 축이 **가변 / 불변**.

| 타입      | 가변?  | 순서?                              |
| ------- | ---- | -------------------------------- |
| `str`   | ❌ 불변 | ✅ 시퀀스                            |
| `list`  | ✅ 가변 | ✅ 시퀀스                            |
| `tuple` | ❌ 불변 | ✅ 시퀀스                            |
| `range` | ❌ 불변 | ✅ 시퀀스 느낌 (연속 정수)                 |
| `dict`  | ✅ 가변 | (파이썬3.7+에서 입력 순서 기억하지만 본질은 키 기반) |
| `set`   | ✅ 가변 | ❌ 순서 없음, 중복 없음                   |

이 전체 그림을 머리에 깔고, 하나씩 들어가면 헷갈림이 훨씬 줄어든다.

## 2. 리스트(list)

### 2-1. 리스트란?

* 여러 개의 값을 **순서 있게** 저장하는, **가변**(mutable) 시퀀스 타입
* `[]` 대괄호 사용

  ```python
  my_list = [1, 2, 3]
  my_list2 = ['a', 'b', 'c']
  my_list3 = [1, 2, 3, 'python', ['hello', 'world']]
  ```

### 2-2. 시퀀스로서의 리스트

시퀀스 공통 기능을 모두 쓸 수 있다.

* 인덱싱: `my_list[0]`, `my_list[-1]`
* 슬라이싱: `my_list[1:3]`
* 길이: `len(my_list)`
* 포함 여부: `x in my_list`
* 반복: `for x in my_list: ...`

### 2-3. 중첩 리스트

리스트 안에 리스트가 들어간 형태.

```python
nested = [1, 2, ['a', 'b', 'c']]
nested[2]      # ['a', 'b', 'c']
nested[2][1]   # 'b'
```

인덱스를 **두 번** 쓰면 2차원 배열처럼 사용할 수 있다.

### 2-4. 리스트의 가변성

가변이라는 건 “값을 바꾸거나, 추가/삭제해도 **새 리스트를 만드는 게 아니라 기존 걸 수정**한다”는 의미.

```python
my_list = [1, 2, 3]
my_list[0] = 100     # 요소 수정
my_list.append(4)    # 뒤에 추가
del my_list[1]       # 요소 삭제
```

그래서 리스트는 “바뀔 수 있는 시퀀스”라는 느낌으로 기억하면 된다.

---

## 3. 튜플(tuple)

### 3-1. 튜플이란?

* 여러 값을 순서대로 저장하지만, **불변(immutable)** 시퀀스
* `()` 소괄호 사용 (혹은 그냥 콤마로만도 튜플이 됨)

  ```python
  my_tuple = (1, 2, 3)
  my_tuple2 = 1, 2, 3   # 괄호 없어도 콤마 기준이면 튜플
  single = (1,)         # 요소가 하나일 땐 콤마 필수
  ```

### 3-2. 시퀀스 기능은 리스트와 비슷

* 인덱싱, 슬라이싱, `len()`, `in` 다 사용 가능
* 다만 **값 자체를 직접 수정할 수 없다**는 차이만 있다.

```python
my_tuple = (1, 2, 3)
# my_tuple[0] = 10  # 에러: 'tuple' object does not support item assignment
```

### 3-3. 왜 쓸까?

* “이 값들은 바뀌면 안 된다”는 의미를 코드에 드러낼 때
* 여러 값을 한 번에 리턴할 때 `(x, y)` 형태로 많이 사용
* 딕셔너리의 키나 set의 원소처럼 “불변이어야 하는 곳”에 쓰기 좋다.

## 4. range

### 4-1. range란?

* **연속된 정수 시퀀스**를 표현하는, 불변 타입
* `range(시작, 끝, 간격)` 형태

  * `시작` 기본값: 0
  * `간격(step)` 기본값: 1
  * `끝`은 **포함되지 않는다** → `range(0, 5)`는 `0,1,2,3,4`

```python
range(5)          # 0,1,2,3,4
range(1, 5)       # 1,2,3,4
range(1, 10, 2)   # 1,3,5,7,9
range(10, 0, -2)  # 10,8,6,4,2
```

* 주로 `for` 반복문에서 인덱스용 숫자 시퀀스 만들 때 쓴다.
* 실제로는 “게으른 숫자 생성기” 느낌이고, 필요하면 `list(range(...))`로 리스트로 바꿔 쓸 수 있다.

## 5. 딕셔너리(dict)

### 5-1. 딕셔너리란?

* `key: value` 짝으로 저장하는 **매핑 타입**
* **순서가 아니라 키를 기준으로 값을 찾는 구조**
* `{}` 중괄호 사용

```python
person = {
    'name': '홍길동',
    'age': 25,
    'city': 'Seoul',
}
```

### 5-2. 키와 값 규칙

* **Key**

  * 중복되면 안 됨
  * **불변 타입만** 가능 (str, int, float, tuple 등)
* **Value**

  * 어떤 타입이든 가능 (list, dict, 함수까지 다 가능)

### 5-3. 값 접근

```python
person['name']     # '홍길동'
person.get('age')  # 25
```

* `[]`로 접근할 때 없는 키면 에러
* `get()`은 없는 키면 `None` 또는 지정한 기본값을 반환

```python
person.get('job', 'unemployed')
```

## 6. 세트(set)

### 6-1. 세트란?

* **순서가 없고, 중복이 없는** 가변 컬렉션
* 수학에서 말하는 “집합” 그 자체 느낌
* `{1, 2, 3}`처럼 쓰지만, **빈 세트는 `set()`** (빈 `{}`는 딕셔너리)

```python
my_set = {1, 2, 3}
my_set2 = {1, 2, 2, 3} # 실제 값은 {1,2,3}
```

### 6-2. 집합 연산

세트의 핵심 기능은 **집합 연산**.

* 합집합: `a | b` 또는 `a.union(b)`
* 교집합: `a & b` 또는 `a.intersection(b)`
* 차집합: `a - b`
* 대칭 차집합(둘 중 한쪽에만 있는 것): `a ^ b`

```python
a = {1, 2, 3}
b = {3, 4, 5}

a | b   # {1,2,3,4,5}
a & b   # {3}
a - b   # {1,2}
a ^ b   # {1,2,4,5}
```

* 리스트의 중복을 제거하고 싶을 때 `set()`을 많이 쓴다.

## 7. 기타 타입: None / Boolean

### 7-1. None

* “값이 없다”라는 특별한 값
* 타입 이름도 `NoneType`, 값도 `None` 하나뿐인 **싱글톤**
* 보통 “아직 아무 것도 정해지지 않았음”을 표현할 때 사용

```python
my_var = None
if my_var is None:
    ...
```

### 7-2. Boolean

* 참 / 거짓을 표현하는 타입
* 값은 `True`, `False` 두 개만 있다.
* 조건문, 반복문에서 핵심

```python
is_active = True
is_logged_in = False
```

## 8. 가변(Mutable) vs 불변(Immutable)

핵심 질문: **“이 객체를 만든 뒤, 내부 내용을 바꿀 수 있는가?”**

* **불변(Immutable)**: 내용 자체를 수정할 수 없음

  * `str`, `tuple`, `range`
* **가변(Mutable)**: 내용 수정, 추가, 삭제 가능

  * `list`, `dict`, `set`

예시 비교:

```python
# 문자열: 불변
my_str = 'hello'
# my_str[0] = 'H'  # 에러

# 리스트: 가변
my_list = [1, 2, 3]
my_list[0] = 100
```

이 개념이 중요한 이유:

* 함수 인자로 넘길 때, 가변 객체는 원본이 바뀔 수 있다.
* 딕셔너리의 key, set의 요소에는 **불변 타입만** 올 수 있다.

## 9. 형변환(Type Conversion)

### 9-1. 형변환이란?

* 어떤 타입을 **다른 타입으로 바꾸는 것**
* 예: 문자열 `'100'` → 정수 `100`, 실수 `3.14` → 정수 `3`

### 9-2. 암시적 형변환 (Implicit)

* 파이썬이 알아서 형을 바꾸는 경우
* 주로 **정수 + 실수** 같이 숫자끼리 섞일 때 일어난다.

```python
result = 3 + 3.5   # 3이 3.0으로 바뀌어서 6.5 (float)
```

* 서로 다른 타입이 섞였을 때, **손실이 덜한 쪽(더 큰 범위)**으로 바꾸는 느낌

### 9-3. 명시적 형변환 (Explicit)

* 내가 직접 `int()`, `float()`, `str()` 같은 함수를 써서 바꾸는 것

대표 함수들:

* `int(x)` : 정수형으로
* `float(x)` : 실수형으로
* `str(x)` : 문자열로
* `list(x)` : 시퀀스를 리스트로
* `tuple(x)` : 시퀀스를 튜플로
* `set(x)` : 중복 제거된 집합으로

주의할 점:

```python
int('123')   # 123 (OK)
int('3.14')  # 에러 (실수 문자열은 바로 int 안됨)
float('3.14')# 3.14 (OK)
```

## 10. 연산자 정리

### 10-1. 산술 연산자

* `+` 더하기
* `-` 빼기
* `*` 곱하기
* `/` 나누기 (항상 float)
* `//` 몫
* `%` 나머지
* `**` 거듭제곱

```python
7 / 2   # 3.5
7 // 2  # 3
7 % 2   # 1
2 ** 3  # 8
```

### 10-2. 복합 대입 연산자

`a = a + b`를 줄여서 쓰는 표현들

* `a += b`
* `a -= b`
* `a *= b`
* `a /= b`
* `a //= b`
* `a %= b`
* `a **= b`

가독성을 위해 많이 쓰이므로 눈에 익혀두기.

### 10-3. 비교 연산자

* `==` 같다
* `!=` 다르다
* `>` 크다
* `<` 작다
* `>=`, `<=`
* `is`, `is not` : “같은 객체인가?” (정체성 비교)

중요 포인트:
**값 비교는 `==`, 객체 자체 비교는 `is`**

```python
a = [1, 2]
b = [1, 2]

a == b      # True (값이 같다)
a is b      # False (서로 다른 리스트 객체)

x = None
x is None   # 이런 식으로 is는 주로 None, True, False 같은 싱글톤에 사용
```

### 10-4. 논리 연산자

* `and` : 둘 다 True여야 True
* `or`  : 둘 중 하나만 True여도 True
* `not` : 진리값 반전

```python
True and False  # False
True or False   # True
not True        # False
```

파이썬의 `and`, `or`는 **단축 평가**도 한다 → 앞에서 이미 결과가 결정되면 뒤는 계산 안 함.

### 10-5. 멤버십 연산자

* `in`, `not in`
* 어떤 값이 시퀀스/컬렉션 안에 들어있는지 확인

```python
3 in [1,2,3]       # True
'a' in 'apple'     # True
'key' in {'key':1} # True (딕셔너리는 key 기준)
```

### 10-6. 시퀀스 연산자

* `+` : 시퀀스 이어 붙이기
* `*` : 시퀀스 반복

```python
[1, 2] + [3, 4]  # [1,2,3,4]
'hi' * 3         # 'hihihi'
```

### 10-7. 연산자 우선순위

(대략적으로 중요한 것만)

1. 괄호 `()`
2. 지수 `**`
3. 단항 연산자 `+x`, `-x`, `not x`
4. 곱셈 / 나눗셈 / 몫 / 나머지 `*`, `/`, `//`, `%`
5. 덧셈 / 뺄셈 `+`, `-`
6. 비교, `in`, `is`
7. `not`
8. `and`
9. `or`

실제로는 헷갈리면 **그냥 괄호로 명확하게 묶어주는 습관**이 제일 안전하다.

## 11. 오늘 TIL 요약

* 파이썬의 컬렉션은 `list/tuple/range/dict/set`이고,
  **“순서 vs 키/집합” + “가변 vs 불변”** 두 축으로 구조를 이해하면 훨씬 덜 헷갈린다.
* `list`는 가변 시퀀스, `tuple`·`range`는 불변 시퀀스,
  `dict`는 키-값 매핑, `set`은 중복 없는 집합.
* `None`은 “값 없음” 싱글톤, 불리언은 `True/False`.
* 형변환은 암시적(숫자끼리 자동)과 명시적(`int`, `float`, `str` 등)으로 나뉜다.
* 연산자는 산술/비교/논리/멤버십/시퀀스 등으로 나뉘고,
  `==` vs `is`, `and`/`or`의 단축 평가, 우선순위를 함께 이해하는 게 중요하다.
