
## 1. ***프로그래밍***과 ***프로그램***

* **프로그래밍이란**

  * 컴퓨터에게 일을 “시키기 위해” **명령을 설계하고 작성하는 과정**.
  * 사람의 생각(문제 해결 아이디어)을 컴퓨터가 이해할 수 있는 **명령어의 순서**로 바꾸는 작업.

* **프로그램(Program)이란**

  * **“문제를 해결하기 위한 명령어들의 집합”**.
  * 예시 느낌

    * 라면 끓이기 순서, 카페에서 주문 절차처럼
      “1 → 2 → 3 단계”로 정리된 절차가 곧 프로그램의 개념이라고 보면 됨.
  * 프로그래밍의 목표

    * 같은 문제를 다시 만났을 때, **정확하고 일관된 방식으로 자동 처리**할 수 있게 만드는 것.

* **프로그래밍이 어려운 이유**

  * 사람 입장에서 “당연한” 것들도 컴퓨터에게는 **하나하나 명확한 규칙**으로 말해줘야 하기 때문.
  * 작은 오타, 순서 하나만 잘못돼도 **전혀 다른 동작**을 하거나 에러가 남.

## 2. ***프로그래밍 언어***와 ***Python***

* **프로그래밍 언어**

  * 사람과 컴퓨터가 소통하기 위한 **형식적인 언어**.
  * 우리가 쓰는 자연어(한국어, 프랑스어)가 아니라,
    컴퓨터가 이해할 수 있도록 규칙이 엄격하게 정해져 있는 언어.
  * 역할

    * **생각을 코드 형태로 표현**하게 해줌.
    * 같은 문제를 다른 사람/컴퓨터와 **공유·재사용**할 수 있게 함.

* **Python 소개**

  * 컴퓨터와 직접 상호작용하고 **문제를 해결하는 도구**로서의 언어.
  * Python을 배우는 이유(요약)

    * 문법이 비교적 **간단하고 읽기 쉬워서** 입문자에게 적합.
    * 웹, 데이터분석, 인공지능, 자동화 등 **다양한 분야에서 사용**됨.
    * 인공지능/머신러닝 관련 라이브러리·프레임워크에서 **가장 널리 사용되는 언어 중 하나**라서 활용성이 높음.

* **Python 실행 방식(그림 흐름)**

  * 우리가 작성한 **Python 소스코드** →
    Python **인터프리터**가 한 줄씩 읽고 해석 →
    그 결과가 컴퓨터에서 실행됨.
  * 즉, “코드를 쓰고 저장 → 인터프리터가 읽어서 실행” 구조라고 이해하면 됨.

## 3. ***표현식(Expression)***과 ***값(Value)***

* **표현식(Expression)**

  * **“하나의 값으로 평가될 수 있는 모든 코드”**.
  * 예시

    * `3`
    * `3 + 5`
    * `"안녕하세요"`
  * 공통점: **실행하면 어떤 값 하나로 바뀔 수 있다**.

* **값(Value)**

  * 표현식을 실제로 “계산/평가”했을 때 나오는 **결과**.
  * 예시

    * 표현식: `3 + 5` → 값: `8`
    * 표현식: `"hello".upper()` → 값: `"HELLO"`

* **표현식 · 평가 · 값의 관계**

  * **표현식**은 “계산하라고 적어놓은 식”.
  * 그 표현식을 **평가(evaluate)** 하면 **값**이 나온다.
  * 이 값은 다시 또 다른 표현식의 일부로 사용될 수 있다.
    (값 → 다른 계산에 다시 투입 가능)

## 4. ***변수(Variable)***와 ***메모리***

* **변수란?**

  * **“값을 나중에 다시 사용하기 위해, 그 값에 붙여주는 고유한 이름”**.
  * 어떤 값을 메모리에 저장해 두고, 그 위치를 **이름으로 가리키는 기능**.

* **변수 할당(Variable Assignment)**

  * 기본 형태:

    ```python
    변수이름 = 표현식
    ```
  * 예시

    ```python
    degrees = 36.5
    ```

    * 오른쪽 `36.5`라는 값을 먼저 계산(평가)하고
      왼쪽 이름 `degrees`가 그 값을 가리키도록 메모리에 기록.

* **변수 이름 규칙(대표적인 것들)**

  * 영문자, 숫자, 밑줄(_)로 구성.
  * **숫자로 시작할 수 없음**.
  * `for`, `if`, `while` 같은 **예약어는 변수 이름으로 사용 불가**.
  * (이 규칙을 어기면 문법 에러 발생.)

* **재할당(Reassignment)**

  * 이미 값이 들어 있는 변수에 **새로운 값을 다시 넣는 것**.
  * 예시

    ```python
    degrees = 36.5
    degrees = 'hello'
    ```

  * 같은 이름 `degrees`지만, **이름이 가리키는 값이 바뀜**.
  * 메모리 관점

    * 처음에는 `degrees` → 36.5
    * 이후에는 `degrees` → `'hello'`
    * “이름 표지판은 그대로인데, 가리키는 대상이 바뀐다”라고 이해하면 편함.

* **변수와 메모리 그림 느낌**

  * 메모리에 여러 칸(상자)이 있고, 각 칸에 값이 저장된다.
  * 변수 이름은 그 칸(값)을 가리키는 **화살표(참조)** 역할을 한다.

## 5. ***데이터 타입(Data Types)***의 개념

* **타입(Type)이란?**

  * 변수나 값이 **가질 수 있는 데이터의 종류**.
  * 어떤 타입인가에 따라

    * **어떤 연산이 가능한지**
    * 어떻게 저장되고 다뤄지는지가 달라진다.

* **타입이 필요한 이유**

  * `1 + 2`는 가능하지만 `"hello" + 2`는 오류인 것처럼,
    **데이터 종류에 따라 가능한 동작이 다르기 때문**.
  * 타입을 구분해 두어야

    * 실수 방지
    * 프로그램이 더 명확하고 안정적으로 동작.

* **대표적인 타입 분류**

  * **Number Types**: `int`, `float`, `complex` 등 숫자 관련
  * **Text Sequence Type**: `str` (문자열)
  * **Sequence Types**: 리스트, 튜플, range 등 “여러 값을 순서대로 모아놓은 것”
  * **Non-sequence Types**: 집합, 딕셔너리, Boolean, None, 함수 등

## 6. ***정수(int)***와 ***실수(float)***

* **정수 자료형(int)**

  * **소수점이 없는 숫자**를 표현.
  * 예시

    ```python
    student_count = 30
    temperature = -5
    balance = 0
    ```

* **실수 자료형(float)**

  * **소수점이 있는 더 정밀한 숫자**를 표현.
  * 예시

    ```python
    pi = 3.14
    weight = 65.5
    tax_rate = 0.1
    ```

  * 매우 크거나 작은 수는 **지수 표기법**으로도 표현 가능
    (예: `1.23e9`, `3.14e-3` 형태의 표현이 등장).

* **숫자형의 행동: 산술 연산**

  * 기본 연산자

    * `+` : 더하기
    * `-` : 빼기
    * `*` : 곱하기
    * `/` : 나누기(실수 결과)
    * `//` : 몫
    * `%` : 나머지
    * `**` : 거듭제곱
  * 연산자 우선순위가 있어서, 괄호를 적절히 사용해야
    원하는 순서대로 계산됨.

## 7. ***시퀀스 타입(Sequence Types)***과 ***문자열(str)***

* **시퀀스 타입이란**

  * 여러 개의 값을 **순서대로 나열해 저장하는 자료형**.
  * 각 값은 **인덱스(위치 번호)**를 통해 접근 가능.
  * 문자열, 리스트, 튜플, range 등이 여기에 해당.

* **문자열(str)이란**

  * **문자들의 순서가 있는, 변경 불가능한 시퀀스 자료형**.
  * 글자들의 시퀀스, 텍스트 데이터를 다룰 때 사용.

* **문자열 만들기**

  * 작은따옴표 `'...'` 또는 큰따옴표 `"..."`로 감싸서 생성.
  * 여러 줄 문자열은 따옴표 3개(`'''...'''` 또는 `"""..."""`)로 표현.
  * 예시

    ```python
    greeting = 'Hello, World'
    print(greeting)
    ```

* **이스케이프 시퀀스(Escape Sequence)**

  * 문자열 안에서 **특수한 기능을 하는 문자 조합**.
  * 대표 예

    * `\n` : 줄 바꿈
    * `\t` : 탭
    * `\'` : 작은따옴표 문자
    * `\"` : 큰따옴표 문자
    * `\\` : 역슬래시 자체

* **f-string**

  * 문자열 안에 **변수나 표현식의 결과를 간단하게 삽입**하는 방법.
  * 사용 방법

    ```python
    name = '홍길동'
    age = 25
    greeting = f'안녕하세요, 제 이름은 {name}이고 나이는 {age}살입니다.'
    ```

  * `{}` 안에 변수 이름이나 간단한 표현식을 넣을 수 있음.
  * “문자열 + 값 이어 붙이기”를 보다 **읽기 쉬운 형태**로 만들어 줌.

## 8. ***문자열과 시퀀스 특징*** (인덱싱, 슬라이싱, 불변성)

* **인덱스(Index)**

  * 시퀀스 자료에서 각 값의 **위치 번호**.
  * 문자열에서

    * 첫 글자는 인덱스 `0`
    * 마지막 글자는 인덱스 `-1`처럼 **음수 인덱스로 뒤에서부터** 접근도 가능.
  * 예시 느낌

    * `"hello"`

      * `h` → 0
      * `e` → 1
      * `o` → -1

* **슬라이싱(Slicing)**

  * 시퀀스의 **일부 구간을 잘라서 새로운 시퀀스를 만드는 것**.
  * 기본 형식

    ```python
    some_str[start:end]      # start부터 end-1까지
    some_str[start:end:step] # step 간격으로
    ```

  * 문자열에서도 슬라이싱을 통해 **부분 문자열**을 얻을 수 있음.

* **문자열의 불변성(Immutability)**

  * 문자열은 **한 번 생성되면 그 자체를 바꿀 수 없음**.
  * 예:

    * `"hello"[0] = 'H'` 처럼 직접 글자를 바꾸는 것은 불가능(에러).
  * 바꾸고 싶다면

    * 기존 문자열을 활용해 **새로운 문자열을 만들어야** 한다.
    * 예시 느낌:
      원본 `"hello"`는 그대로 두고,
      `"H" + "ello"` 같은 식으로 새로운 문자열을 만들어 사용.

## 9. 오늘 정리 포인트(나중에 볼 때용 요약)

* ***프로그래밍*** = 문제 해결 아이디어를 컴퓨터가 이해할 수 있는 **명령어 순서**로 옮기는 일.
* ***프로그램*** = 그 명령어들을 모아둔 **집합**.
* ***프로그래밍 언어*** = 사람과 컴퓨터가 소통하기 위한 **형식 언어**, Python은 그중 하나.
* ***표현식***은 **값으로 평가될 수 있는 코드**, ***값***은 그 평가 결과.
* ***변수***는 값을 가리키는 **이름표**, `이름 = 표현식` 형태로 **할당**, 나중에 **재할당** 가능.
* ***데이터 타입***은 값의 종류와 가능한 연산을 정하는 **분류 체계**.
* 숫자형에서는 ***int***(정수), ***float***(실수)를 중심으로 산술 연산을 한다.
* ***시퀀스 타입***은 여러 값을 순서대로 저장하는 자료형이고, ***문자열(str)***은 그 중 **변경 불가능한 문자 시퀀스**.
* 문자열은 따옴표로 만들고, 이스케이프 시퀀스로 특수 문자/줄바꿈을 표현한다.
* ***f-string***으로 문자열 안에 변수/표현식 결과를 깔끔하게 삽입할 수 있다.
* 문자열은 시퀀스라서 **인덱싱, 슬라이싱**이 가능하지만, **불변성** 때문에 내용 자체를 직접 바꾸지는 못한다(새 문자열을 만들어야 함).
